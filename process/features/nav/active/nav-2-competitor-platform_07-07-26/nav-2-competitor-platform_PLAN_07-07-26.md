## Overview

Two new free-text fields on a lead:

- `currentPlatform` — the competitor ticketing platform the organizer currently uses (e.g.
  "Eventbrite", "Luma", "Facebook Events"). Free text, NOT the existing `platform` enum.
- `competitorNotes` — free-text AE pitch notes about our competitive advantage.

Both nullable, both optional everywhere. Nothing about existing lead creation may break.

**Critical distinction:** the existing `crm_leads.platform` column is a `LEAD_PLATFORMS` **enum**
describing the event-source platform. `currentPlatform` is unrelated free text — it MUST be
`text()` in the schema and `z.string().optional()` in zod, **never** the enum.

## Goals

1. Persist both columns (nullable) with a clean additive migration `0030`.
2. Read path: `currentPlatform` badge on Unassigned Leads cards (only when set).
3. Write path: `competitorNotes` editable on lead detail; both fields editable in the edit modal
   and settable on lead creation; `currentPlatform` accepted from the scraper ingest endpoint.
4. `bun run check` + `bun run lint` exit 0; unit tests green.

## Scope

**In scope:** schema + migration, `Lead`/`CreateLeadInput` types, three zod schemas, db mapper +
create/update, unassigned card badge, lead-detail notes textarea, edit modal fields, lead-create
form field, ingest schema + handler.

**Out of scope:** analytics/reporting on competitor platform, dropdown/enum-ifying
`currentPlatform` (kept free text per issue), filtering leads by competitor platform.
